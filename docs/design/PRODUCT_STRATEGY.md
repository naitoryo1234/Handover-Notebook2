## 1. プロダクトビジョン

### コンセプト
**「汎用コア + 業態プリセット」の分離型SaaS**

- 顧客管理と予約管理を**独立モジュール**として提供
- 片方だけでも使える設計（既存システムとの共存を想定）
- 業態ごとのプリセットを適用することで、業界特化型に変化

### ターゲット
**個人経営の施術院・サロン・スクール（ひとり社長）**

- 従業員0〜3名規模
- 既存のレセコン/電カルを持っているケースも多い
- 月額¥5,000以上の大手SaaSは「過剰」と感じている層

---

## 2. 市場ポジショニング

### レッドオーシャンの隙間を狙う

```
[大手SaaS: Airリザーブ、サロンボード等]
  月額 ¥5,000〜¥30,000
  機能豊富だが過剰 / カスタム不可
        ↓↓↓ ここを狙う ↓↓↓
[Business Notebook]
  月額 ¥1,980〜¥2,980
  必要な機能だけ / 業態プリセットでフィット
        ↑↑↑ ここを狙う ↑↑↑
[紙 / Excel / Googleカレンダー]
  無料だが情報が散らばる
```

### なぜ戦えるのか

1. **AI支援開発による低コスト開発**
   - 3日でプロトタイプ完成レベルの生産性
   - 業態プリセット追加も1-2日で可能

2. **分離型設計**
   - 「顧客管理だけ」「予約管理だけ」のニーズに対応
   - 既存システム（レセコン等）との共存が可能

3. **安価な個別カスタム**
   - 大手がやらない価格帯・規模感
   - 「うちだけの機能が欲しい」に低コストで対応可能

## 3. 差別化ポイント

### キラー機能: 音声入力 + AI整形

> **「話すだけで、カルテが整う。」**

| 機能 | 競合状況 |
|:---|:---|
| 音声入力 | ほぼ未実装 |
| AI整形・要約 | 未実装 |
| モバイル対応 | 一部のみ |
| **3点セット** | **Business Notebookのみ** |

**これがあるから選ばれる**、というレベルで磨き込む価値がある。

### 活用シナリオ

```
施術終了
  ↓
スマホに向かって話す
「今日は腰の張りが強かったので重点的にほぐしました。
 次回は2週間後で」
  ↓
AIが整形して保存
  ↓
次の患者へ
```

**手が空かない・時間がない問題を解決。**

## 5. 業態プリセット戦略

### 仕組み

```
[汎用コア]
  ├─ 顧客管理基盤
  ├─ 予約管理基盤
  └─ 共通機能（認証、検索、タイムライン）

[業態プリセット]
  ├─ デフォルトタグセット
  ├─ 追加フィールド（attributes JSON）
  ├─ AIプロンプトのカスタマイズ
  └─ 機能ON/OFF設定
```

### 既存の技術的土台

| 機能 | 活用方法 |
|:---|:---|
| `tags` (JSON) | 業態ごとのデフォルトタグ |
| `attributes` (JSON) | 業態固有の追加フィールド |
| Feature Flag | 業態ごとの機能ON/OFF |
| `SystemSetting` | プリセット設定値の保存 |

### プリセット候補（将来）

| 業態 | 想定される追加機能 |
|:---|:---|
| 鍼灸院・整体院 | 禁忌警告、施術部位画像、回数券 |
| ネイルサロン | デザインギャラリー、次回提案 |
| 個人塾・スクール | 科目別進捗、保護者連絡 |

## 6. 開発における原則

### AI支援開発の活用

- **ドラフト品質を高速生産** → 人間がレビュー
- 「速く作る」と「品質を上げる」は別フェーズで意識する
- 失敗のコストが低いので、実験回数を増やす

### 判断基準

機能追加の際は以下を問う：

1. **ひとり社長が本当に使うか？**
2. **既存システム（レセコン等）と競合しないか？**
3. **音声入力/AI整形と組み合わせて価値が出るか？**
