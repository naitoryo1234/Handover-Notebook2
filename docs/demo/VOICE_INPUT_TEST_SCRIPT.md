# 音声入力テスト台本

鍼灸院向けデモでの音声入力機能テスト用シナリオです。

---

## 🎯 Customer Notebook（顧客詳細画面）

### テストシナリオ1: 基本的な施術記録

**話す内容:**
```
今日来た山田さんなんですけど、腰が痛いって言ってて、
昨日重いものを持ったらしいです。
右の腰あたりが特に痛いみたいで、
鍼を中心に施術しました。
次は1週間後に来てもらうことにしました。
```

**期待される整形結果:**
```
【主訴】腰痛（右側）
【経緯】昨日重いものを持った際に発症
【施術】鍼を中心に施術を実施
【次回】1週間後に再来院予定
```

---

### テストシナリオ2: 禁忌情報を含む記録

**話す内容:**
```
今日の鈴木さんなんですが、首と肩のこりがひどいということで来院されました。
えーと、ペースメーカーを入れているので電気は使わないでください。
今日は指圧とお灸で対応しました。
肩甲骨周りが特に張っていて、次回は2週間後の予定です。
```

**期待される整形結果:**
```
【主訴】首・肩のこり
【禁忌】ペースメーカー装着のため電気療法不可
【施術】指圧・灸を実施
【所見】肩甲骨周囲に著明な緊張あり
【次回】2週間後
```

---

### テストシナリオ3: 複数の訴え

**話す内容:**
```
佐藤さんですが、膝が痛いのと、腰も張るって言ってました。
特に階段を降りる時が痛いらしくて。
今日は電気と手技で腰から膝にかけてやりました。
あと、自宅でできるストレッチも教えました。
次回は来週の水曜日で予約しました。
```

**期待される整形結果:**
```
【主訴】膝痛・腰部の張り
【詳細】階段下降時に膝痛増強
【施術】腰部〜膝部に電気療法・手技療法を実施
【指導】自宅でのストレッチ指導
【次回】来週水曜日
```

## 🎯 Reservation V2（予約管理）音声コマンド

### テストシナリオA: 名前検索

**話す内容**: 「田中さん」「山田さんの予約」

**期待される動作**: 「田中」「山田」で顧客検索

---

### テストシナリオB: 日付フィルター

**話す内容**: 「明日の予約」「来週の火曜日」

**期待される動作**: 指定日にカレンダー移動

---

### テストシナリオC: 時間帯フィルター

**話す内容**: 「午前中」「17時以降」「18時周辺」

**期待される動作**: 時間帯でフィルタリング

---

### テストシナリオD: ステータスフィルター

**話す内容**: 「担当未定」「申し送り」

**期待される動作**: 未割り当て/未確認申し送りでフィルタリング

---

### テストシナリオE: 複合コマンド

**話す内容**: 「高橋先生の明日の予約」

**期待される動作**: スタッフ「高橋」＋明日でフィルタリング

---

## 📝 デモ時の注意点

1. **話すタイミング**: マイクボタンを押してから話し始める
2. **話す速度**: 普通〜やや速め（自然な速度でOK）
3. **フィラー**: 「えーと」「あの」は自動で削除される
4. **専門用語**: 正確に発音する（例: 仙腸関節、鍼通電）
5. **HTTPS必須**: Vercelデプロイ版でテスト
