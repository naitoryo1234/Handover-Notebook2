## 1. 当日運用（5件）

### M-UC-01: 今日の予約一覧を確認する

- **想定ユーザー**: 院長 / スタッフ
- **利用シーン（モバイル状況）**: 出勤直後、電車内で片手操作、開院5分前に立ったまま確認
- **目的**: 今日の予約全体を把握し、準備すべき顧客を確認する
- **基本フロー**:
  1. アプリを起動（またはホームをタップ）
  2. 「今日の予約」が時系列で表示される
  3. 各カードで顧客名・時間・担当を確認
  4. 必要に応じてカードタップで顧客詳細へ
- **入力方法の想定**: タップのみ（入力なし）
- **例外・失敗パターン**:
  - 今日の予約が0件 → Empty State表示「本日の予約はありません」
  - 圏外で読み込み失敗 → 最終取得データを表示 + 「オフライン」バッジ
- **必要な画面/コンポーネント候補**:
  - 今日の予約リスト（カード形式）
  - 予約カード（顧客名・時間・ステータスバッジ）
  - 顧客詳細へのクイックナビ
- **必要なデータ/入力**: 予約一覧（当日フィルタ）、顧客基本情報
- **受け入れ条件（Done）**:
  - [ ] アプリ起動から2タップ以内で今日の予約一覧が見れる
  - [ ] 予約0件時にEmpty Stateが表示される

### M-UC-02: 次の顧客の情報を事前確認する

- **想定ユーザー**: スタッフ
- **利用シーン（モバイル状況）**: 施術の合間（2〜3分の隙間）、片手でスマホを持ちながら
- **目的**: 次の顧客の過去履歴・注意事項を把握し、スムーズな対応準備をする
- **基本フロー**:
  1. 今日の予約リストを開く
  2. 「次の予約」が自動ハイライト or 上部固定
  3. カードをタップして顧客サマリ表示
  4. 重要メモ・前回記録をスクロールで確認
- **入力方法の想定**: タップのみ
- **例外・失敗パターン**:
  - 次の予約がない（最後の施術後）→「本日の予約は終了しました」
  - 新規顧客で履歴なし →「初回来店」バッジ表示
- **必要な画面/コンポーネント候補**:
  - 「次の予約」ハイライトカード
  - 顧客サマリビュー（コンパクト版）
  - 重要フラグ付きメモの優先表示
- **必要なデータ/入力**: 次の予約情報、顧客詳細、過去記録（重要フラグ優先）
- **受け入れ条件（Done）**:
  - [ ] 1タップで次の顧客情報にアクセスできる
  - [ ] 30秒以内で要点が把握できる

### M-UC-03: 来店した顧客をチェックインする

- **想定ユーザー**: 受付 / スタッフ
- **利用シーン（モバイル状況）**: 受付カウンターで立ったまま、顧客対応しながら片手操作
- **目的**: 顧客の来店を記録し、施術準備状態に移行する
- **基本フロー**:
  1. 今日の予約リストを表示
  2. 来店した顧客の予約カードを特定
  3. 「チェックイン」ボタンをタップ（スワイプでも可）
  4. ステータスが「来店済み」に更新される
- **入力方法の想定**: 1タップ or 右スワイプ
- **例外・失敗パターン**:
  - 予約なしで来店（飛び込み）→「予約なし来店」フロー誘導
  - 誤タップ →「取り消す」ボタンを3秒間表示
- **必要な画面/コンポーネント候補**:
  - 予約カードのスワイプアクション
  - チェックイン確認トースト（Undo付き）
  - ステータスバッジ更新アニメーション
- **必要なデータ/入力**: 予約ID、ステータス更新
- **受け入れ条件（Done）**:
  - [ ] 1タップまたは1スワイプでチェックイン完了
  - [ ] 誤操作時に3秒以内なら取り消し可能

### M-UC-04: 遅刻連絡を受けて予約時間を変更する

- **想定ユーザー**: 受付 / 院長
- **利用シーン（モバイル状況）**: 電話対応中、片手でスマホ操作、30秒以内に完了したい
- **目的**: 顧客からの遅刻連絡に対応し、予約時間を即座に調整する
- **基本フロー**:
  1. 今日の予約から対象顧客を見つける
  2. 予約カードをタップ → 編集モード
  3. 時間のみ変更（ボトムシート or インライン）
  4. 「保存」タップで完了
- **入力方法の想定**: タップ選択（時間ピッカー）
- **例外・失敗パターン**:
  - 変更先時間に別予約あり →「重複警告」を表示、強制保存は可能
  - 電話中に操作ミス → 変更確認ダイアログで防止
- **必要な画面/コンポーネント候補**:
  - 予約編集ボトムシート（時間のみモード）
  - 時間ピッカー（15分刻み推奨）
  - 重複警告トースト
- **必要なデータ/入力**: 予約ID、新しい開始時間
- **受け入れ条件（Done）**:
  - [ ] 3タップ以内で時間変更が完了
  - [ ] 重複がある場合は警告が表示される

### M-UC-05: 当日キャンセルを処理する

- **想定ユーザー**: 受付 / 院長
- **利用シーン（モバイル状況）**: 電話対応中、急ぎで処理、片手操作
- **目的**: 当日キャンセルを記録し、空き枠を把握する
- **基本フロー**:
  1. 今日の予約から対象を選択
  2. 予約カードを長押し or メニューから「キャンセル」
  3. キャンセル理由を選択（任意）
  4. 確認ダイアログで「キャンセル確定」
- **入力方法の想定**: タップ + 選択式（理由）
- **例外・失敗パターン**:
  - 誤ってキャンセル → 確認ダイアログ必須で防止
  - キャンセル後の復元 → 履歴から「予約を復元」可能
- **必要な画面/コンポーネント候補**:
  - キャンセル確認ダイアログ
  - 理由選択（ドロップダウン or チップ）
  - キャンセル完了トースト（Undo付き）
- **必要なデータ/入力**: 予約ID、ステータス更新（cancelled）、キャンセル理由（任意）
- **受け入れ条件（Done）**:
  - [ ] 確認ダイアログなしにキャンセルされない
  - [ ] キャンセル後5秒以内なら取り消し可能

## 2. クイック検索（3件）

### M-UC-06: 顧客を名前で素早く検索する

- **想定ユーザー**: 受付 / スタッフ
- **利用シーン（モバイル状況）**: 電話対応中、顧客が目の前で待っている、5秒以内に見つけたい
- **目的**: 顧客名の一部から即座に顧客情報にアクセスする
- **基本フロー**:
  1. 検索アイコンをタップ（常時表示推奨）
  2. 名前の一部を入力（2文字以上）
  3. インクリメンタル検索で候補表示
  4. 候補をタップして顧客詳細へ
- **入力方法の想定**: テキスト入力（最小2文字）
- **例外・失敗パターン**:
  - 該当なし →「見つかりません」+ 新規登録リンク
  - 同姓同名 → フリガナ・生年月日で識別表示
- **必要な画面/コンポーネント候補**:
  - グローバル検索バー（常時アクセス可能）
  - 検索結果リスト（コンパクト）
  - 顧客識別情報（フリガナ・最終来店日）
- **必要なデータ/入力**: 顧客名（部分一致）、フリガナ
- **受け入れ条件（Done）**:
  - [ ] 2文字入力で検索が開始される
  - [ ] 入力から結果表示まで500ms以内

### M-UC-07: 電話番号から顧客を検索する

- **想定ユーザー**: 受付
- **利用シーン（モバイル状況）**: 電話着信時、発信者番号から顧客を特定したい
- **目的**: 電話番号から顧客を特定し、名前を呼んで対応、履歴を即確認
- **基本フロー**:
  1. 検索バーに電話番号（下4桁でも可）を入力
  2. 該当顧客がヒット
  3. タップして詳細表示
- **入力方法の想定**: 数字入力（4桁以上）
- **例外・失敗パターン**:
  - 未登録番号 →「新規顧客」として登録フローへ誘導
  - 複数ヒット → リストで選択
- **必要な画面/コンポーネント候補**:
  - 検索バー（数字対応）
  - 電話番号ハイライト表示
- **必要なデータ/入力**: 電話番号（部分一致）
- **受け入れ条件（Done）**:
  - [ ] 下4桁で検索可能
  - [ ] 未登録時に新規登録への導線がある

  - 予約作成ボトムシート

## 5. ミス対策（3件）

### M-UC-15: 二重予約を検知して解消する

- **想定ユーザー**: 受付 / 院長
- **利用シーン（モバイル状況）**: 予約入力後に警告が出た、または予約一覧で重複発見

- **受け入れ条件（Done）**:
  - [ ] 重複時は必ず警告が表示される
  - [ ] 重複を解消せずに保存する場合も明示的な操作が必要

### M-UC-16: 誤操作を取り消す（Undo）

- **想定ユーザー**: 全員
- **利用シーン（モバイル状況）**: 誤タップでキャンセルしてしまった、間違った顧客を選んだ
- **目的**: 直前の操作を取り消して元に戻す
- **基本フロー**:
  1. 操作完了後にUndoトーストが表示される（3〜5秒）
  2. 「取り消す」をタップ
  3. 操作が取り消される
- **入力方法の想定**: 1タップ
- **例外・失敗パターン**:
  - Undo期限切れ → 手動で修正案内
  - 連続Undo → 最後の操作のみUndoable
- **必要な画面/コンポーネント候補**:
  - Undoトースト（固定表示、3〜5秒）
  - 取り消し完了フィードバック
- **必要なデータ/入力**: 直前の操作ログ
- **受け入れ条件（Done）**:
  - [ ] 重要操作（キャンセル、削除、ステータス変更）後にUndoトーストが表示される
  - [ ] 3秒以内なら取り消し可能

### M-UC-19: 前回のメモをテンプレートとして利用する

- **想定ユーザー**: スタッフ
- **利用シーン（モバイル状況）**: 定期来店の顧客、毎回同じような内容を記録

- **必要な画面/コンポーネント候補**:
  - 「前回をコピー」ボタン
  - コピー確認ダイアログ（長文時）
- **必要なデータ/入力**: 前回の記録内容
- **受け入れ条件（Done）**:
  - [ ] 前回記録がある場合のみボタン表示
  - [ ] 1タップでコピー完了
