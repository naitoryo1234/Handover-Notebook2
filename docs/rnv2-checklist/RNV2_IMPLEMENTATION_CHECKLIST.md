# RNV2 実装チェックリスト

> **目的**: Reservation Notebook V2を商用SaaS品質に引き上げるための実装チェックリスト
> 
> **最終更新**: 2025-12-18

---

## P0: 必須 (リリースブロッカー)

### ステータス管理
- [ ] **チェックイン（来店確認）ボタン**
  - テーブルの操作列に「チェックイン」ボタンを追加
  - 既存の `checkInAppointment` サービスを活用
  - ステータスを「予定」→「対応中」に変更

- [ ] **完了ボタン**
  - テーブルの操作列に「完了」ボタンを追加
  - ステータスを「対応中」→「完了」に変更

- [ ] **ステータス表示の視覚的改善**
  - 各ステータスに応じたアイコン・色の統一
  - 予定: ⚪ / 対応中: 🟢 / 完了: ✅ / キャンセル: ❌

### 基本機能（実装済み確認）
- [x] 新規予約作成（2カラムモーダル）
- [x] 予約編集
- [x] 予約キャンセル（削除）
- [x] 顧客名検索（ひらがな→カタカナ変換）
- [x] 日付ナビゲーション（今日/明日/全期間）
- [x] 担当者フィルター
- [x] 未割り当てフィルター

## P1: 推奨 (SaaS品質向上)

### 申し送り管理
- [ ] **申し送り解決マーク**
  - 申し送りダイアログに「解決済みにする」ボタン追加
  - 解決者名・解決日時を記録
  - 解決済みの申し送りは薄く表示

### UI整合性
- [ ] **カレンダー選択と全期間モードの排他制御**
  - 「全期間」モード時はカレンダーのハイライトを解除
  - カレンダー日付クリック時は「全期間」モードを解除

### データ連携（実装済み確認）
- [x] 予約→顧客詳細ページへのリンク
- [x] 予約→カルテ記録作成へのリンク
- [x] サイドバー「本日の予定」リスト
- [x] ミニカレンダー
